Reflection Questions

  1. What is the purpose of try/catch?

    The purpose of try/catch is to prevent the program from crashing and to show useful feedback when errors or invalid inputs occur.

  2. What does throw new Error(...) do?

    It manually creates and throws an error when something does not work as intended or when a condition is invalid, even if the JavaScript syntax itself is correct.

  3. Why do we check response.ok in fetch code?

    We check response.ok to make sure the HTTP request was successful before trying to use the returned data. fetch() does not automatically reject promises for HTTP errors like 404.

  4. What is the difference between sync error handling and Promise .catch()?

    Synchronous error handling with try/catch handles errors that happen immediately during code execution, while Promise .catch() handles asynchronous errors that happen later, such as fetch request failures.

  5. What is finally useful for?

    finally is useful for running code that should execute no matter what happens, whether an error occurs or not.

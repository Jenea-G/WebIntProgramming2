Book Prototype Demo

Part 6 - Reflection Questions

  1. What problem do prototypes solve?

    They solve the probleme of redundancy in the code

  2. What is the difference between an own property and a prototype method?

    The own property belogns on the object and will be individual for each object, while a prototype method is shared by all objects

  3. Where does JavaScript look if a method is not found directly on an object?

    It looks into prototype

  4. Why is book1.displayInfo === book2.displayInfo useful evidence?

    It demonstrates that the prototype method is identical for each object and thus shared.

  5. What happened when you changed book1.category?

    The shadow property was created for book1, thus it's category is no longer a shared property and is disconnected from prototype property

  6. Why is it better to put shared behavior on the prototype instead of inside every object?

    Because it will create copy methods for each instance rather than share the same method
